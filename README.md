# News Bias Detector

An AI-powered web application that analyzes political bias, emotional language, missing context, and credibility of news articles. Built for political and current affairs journalism.

Paste any news article URL and the system scrapes the article, searches the web for the same story across multiple sources, analyzes each one independently, and generates a cross-source comparison report with an interactive dashboard.

## Features

- Automatic article scraping from any news URL
- Real-time web search to find the same story across multiple sources
- Independent analysis of each source for political bias, emotional language, missing context, and credibility
- Cross-source comparison with narrative report generated by LLM
- Interactive dashboard with bias charts, credibility scores, radar chart, and ranked summary table
- Sources tab showing evidence and quotes behind every score

## Tech Stack

- Python 3.11
- Streamlit — web application framework
- LangChain + Groq (LLaMA 3.3 70B / LLaMA 3.1 8B) — LLM analysis
- Tavily — real-time web search
- Newspaper4k — article scraping
- Plotly — interactive visualizations

## How to Run Locally

**Prerequisites**
- Python 3.9 or higher
- A Groq API key — free at https://console.groq.com
- A Tavily API key — free at https://tavily.com

**Steps**

Clone the repository:

git clone https://github.com/Aniket-14-07/news-bias-detector.git
cd news-bias-detector

Create and activate a virtual environment:

python -m venv venv
venv\Scripts\activate

Install dependencies: pip install -r requirements.txt

Run the app: streamlit run app.py

Open your browser at http://localhost:8501, enter your API keys in the sidebar, paste a news article URL, and click Analyze.

## Architecture

The application follows a linear pipeline triggered by a single URL input.

The scraping layer uses Newspaper4k to extract article text, title, and metadata from the provided URL. SSL verification is handled explicitly to ensure compatibility across different network environments.

The search layer uses Tavily's API to perform a real-time web search based on the article's core topic, returning results from across the internet rather than a fixed set of sources. This makes the comparison genuinely dynamic — the sources analyzed depend on what is currently being reported, not a hardcoded list.

The analysis layer sends each scraped article through four dedicated LLM prompts — one each for political bias, emotional language, missing context, and credibility. Each prompt requires the model to extract specific evidence from the article text before assigning a score, grounding the analysis in actual content rather than general impressions. Political bias uses the larger LLaMA 3.3 70B model given the complexity of ideological framing detection. The remaining three dimensions use LLaMA 3.1 8B to reduce token consumption without meaningful accuracy loss.

The report layer aggregates all source analyses and generates a narrative comparison report using the larger model, structured as a professional media analysis with sections covering story overview, bias comparison, emotional language patterns, context gaps, most reliable source, and a final verdict.

The visualization layer renders an interactive Plotly dashboard with four bar charts, a radar chart for multi-dimensional source comparison, and a ranked summary table with composite overall scores.

## Limitations

- Designed for political and current affairs articles. Results on sports, entertainment, or technology articles may not be meaningful.
- Analysis scores are AI-generated interpretations and should be treated as a starting point for human review, not definitive conclusions.
- Some news websites block automated scraping. In such cases the source is skipped and the next available result is used.
- Groq's free tier has a daily token limit. Running analysis on more than 3-4 sources consecutively may exhaust the limit.
